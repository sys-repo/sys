# Staged Workspace Pruning Plan

## Goal

Reduce staged deploy workspaces from:

- full workspace copy

to:

- root essentials
- target package
- transitive workspace package dependencies of that target

## Scope

This is package-level pruning only.

Non-goals:

- pruning files within a retained package
- changing the deploy/runtime contract away from a real workspace-shaped root
- changing target build behavior
- changing deploy entry generation

## Why now

This was not cleanly possible before the workspace graph work.

We now have a persisted workspace package graph at:

- `.tmp/workspace.graph.json`

generated by:

- `-scripts/task.prep.ts`
- `@sys/workspace` `Workspace.Prep.run()`

That snapshot is enough for package-level closure selection.

We do not need module/file pruning for this move.

## Current stage seam

Current whole-workspace staging happens in:

- `src/m.cloud/m.DenoDeploy/m.stage/u.executeStage.ts`
- `src/m.cloud/m.DenoDeploy/m.stage/u.materializeWorkspace.ts`

Today the stage path does:

1. resolve target workspace + target package
2. build the target package
3. copy the whole workspace into the stage root
4. patch staged imports / entry files

## Implementation module

Primary implementation should live in:

- `src/m.cloud/m.DenoDeploy/m.stage/`

Likely file seams:

- `u.executeStage.ts`
- `u.materializeWorkspace.ts`
- new helper for closure calculation
- new helper for selective stage-root rewrite

This does not belong in `m.pipeline`.
It is a staging concern.

## Graph source

Use the persisted package graph first.

Read from the source workspace root via:

- `Workspace.Prep.Graph.read(cwd)`

or the underlying snapshot helper if needed.

Do not re-collect a fresh graph during stage unless the snapshot is missing and
there is an explicitly chosen fallback policy.

## Closure rule

Given target package path `P`:

1. include `P`
2. include every package that is a transitive dependency of `P`

Using current persisted edge direction:

- `from` = dependency
- `to` = dependent

So closure walk should repeatedly include:

- any edge where `edge.to === current`
- then add `edge.from`

until fixed point.

## Stage contents

Retain:

- root `deno.json`
- root `imports.json` when present
- root files already required by the existing deploy/stage path
- retained workspace package directories
- target `dist/`
- generated stage entry files

Prune:

- workspace package directories outside the closure

## Required rewrite

After selective materialization, rewrite staged root `deno.json` so:

- `workspace` contains only retained package dirs

This is required for truthfulness.

We must not leave a staged root declaring workspace children that were pruned.

## Suggested work steps

1. Add a closure helper that maps:
   - snapshot graph
   - target relative package dir
   - -> retained package dir set

2. Replace full `copyDir(workspace, root)` with selective materialization:
   - copy root essentials
   - copy retained package dirs

3. Rewrite staged root `deno.json.workspace`

4. Keep existing:
   - target build
   - stage driver import patch
   - entry rendering
   - prepare/deploy path

5. Add tests proving:
   - unrelated workspace package is absent from staged root
   - transitive dependency package is still present
   - staged `deno.json.workspace` is pruned to the retained set
   - staged target still loads as a workspace

## Test seams

Start in:

- `src/m.cloud/m.DenoDeploy/m.stage/-test/-artifact.test.ts`

Likely add:

- one focused fixture with:
  - target app
  - retained lib
  - unrelated lib

Then assert:

- retained lib exists in stage root
- unrelated lib does not
- staged workspace children list is closure-only

## Risks

Main risk is not graph correctness.
Main risk is staged-root truth drift.

Watch for:

- root files silently assumed by deploy path
- staged `deno.json.workspace` not matching copied dirs
- target build writing `dist/` before stage copy and that output not surviving

## Desired outcome

The deploy stage still feels like a real workspace root, but only the truthful
package closure is shipped.
